A Peer Marketplace

made for students, by students

11/2018 - 01/2020

Git Repository

I was walking down the quad last fall watching students throw away perfectly good furniture from their dorms and wondering why they weren't attempting to squeeze value out of what they had. I wanted to help people out, to give them more options with the items they don't need anymore.

I wanted to create a solution, to give my peers a network for specifically selling their used or unused dorm items that are being constantly rebought year after year.

First... I designed a UI that catered to the user in the most trusting way, resembling that of the simple and clean iOS design elements.

Secondly... I enrolled in a few online courses outside of class and spent countless hours learning Swift and becoming familiar with Xcode.

Third... I integrated Google's Firebase so that I can use a realtime database to manage the users saved data, as well as share their listings and communicate with one another.

Lastly... I brought it all together, I put myself in the users' perspective. To allow them to access features that they wouldn't only need but features that would simplify and better their experience.

Mapping Company Info.

loading and fetching three thousand lines of info

10/2019 - 11/2019

Git Repository

For my CIS final project, we were given a list of 3,000+ companies that includes:

  • Company Name
  • Company Symbol
  • Last Sale
  • Marekt Cap
  • IPO Year
  • Sector
  • Industry
  • URL for Summary Quote

I was tasked to create data structures to store the data per company and then fetch the structure once storing from the CSV input file is complete. My goal was to do this as fast as possible, so I implemented functions in order to store and fetch each either linked lists or hashing.

My end project review:

I chose the Double Hash function that uses the return value with prime - (key % prime) and the prime number being 7 because, the nature of prime numbers reduces the common factor between indicies which in turn would reduce the amount of collisions while inserting values into the hashtable. Implementing the linked list to achieve the same results as the hash table showed me that linked lists are not as efficient as I believed going into this project. The time shows that, the linked list took 20 times longer to Store and Fetch the 3,400 companies from the CSV file. Simply because you have to iterate through each node in the linked list to point to the next node which slows down the process immensley.
